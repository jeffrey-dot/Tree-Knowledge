# 路线图

## Phase 1：知识工作台最小闭环
目标是让系统从“一个问题”走到“一个可导航的节点网络雏形”。

范围：
- 创建根节点
- 三栏工作台
- 当前节点切换
- 图谱最小可视化
- 直接生成节点
- 候选节点生成与确认
- 节点详情编辑

完成标准：
- 用户可从根问题扩展出至少 20 个节点
- 用户可在任意祖先节点回跳并继续新分支
- 未确认的候选节点不会污染正式图谱

## Phase 2：结构修正与上下文稳定化
目标是让知识结构可维护，让 LLM 的生成过程稳定可控。

范围：
- 修改主父节点
- 新增和删除关系边
- 节点上下文快照
- 搜索与定位
- 工作台聚合读取优化

完成标准：
- 节点移动父级后，路径、图谱、上下文同步更新
- 搜索结果可直接恢复到对应工作台上下文
- 长节点生成不再依赖整段历史对话

## Phase 3：增强能力
目标是在不破坏核心逻辑的前提下扩展能力。

可选范围：
- 导入 Markdown 或其他笔记
- 更丰富的关系类型
- 图谱视图增强
- 节点合并与拆分
- 更强的候选去重和质量评分

完成标准：
- 新能力不会破坏“节点优先、结构可修正、上下文围绕当前节点”这三条核心原则

## 阶段优先级原则
- 先做结构闭环，再做内容丰富
- 先做当前节点上下文控制，再做全局智能
- 先做稳定可维护的数据模型，再做更复杂的图谱交互
