# 视图地图

## 目标
Tree Knowledge 的桌面应用视图结构必须服务于一个核心原则：  
用户在 `知识库之间切换` 时才离开主工作台，用户在 `节点之间切换` 时应尽量停留在主工作台内完成。

因此，产品不应演化为一个拥有大量分散页面的后台系统，而应当是：
- 一个高感知的桌面启动台
- 一个承载绝大多数操作的主工作台
- 若干围绕工作台展开的辅助视图

## 视图分层
视图按职责分为三层：

### 1. 知识库层
负责展示、选择、创建和管理知识库。

包含视图：
- `桌面启动台`
- `创建/编辑知识库视图`
- `知识库设置视图`
- `Provider 管理视图`

### 2. 节点工作层
负责围绕当前节点进行浏览、生成、编辑和跳转。

包含视图：
- `知识工作台`

### 3. 辅助交互层
负责在不打断主工作台的情况下完成候选确认、搜索和结构编辑。

包含视图或浮层：
- `候选节点确认层`
- `全局搜索层`
- `节点结构编辑层`
- `图谱全览页`（后期）

## 总体视图地图
推荐的 MVP 视图地图如下：

1. `桌面启动台`
2. `创建/编辑知识库视图`
3. `知识工作台`
4. `候选节点确认层`
5. `全局搜索层`
6. `节点结构编辑层`
7. `知识库设置视图`
8. `Provider 管理视图`

后期视图：
- `图谱全览页`
- `导入视图`

## 1. 桌面启动台
### 视图定位
桌面启动台不是简单的列表页，而是 `知识宇宙总览视图`。  
它既要展示所有知识库，又要提供强烈的第一印象，让用户感觉自己进入的是一个正在生长的知识网络，而不是文件夹集合。

### 视图目标
- 展示所有知识库
- 表达每个知识库的活跃度和增长状态
- 帮用户快速回到最值得继续的知识库
- 提供创建新知识库的主入口
- 提供 LLM Provider 设置入口

### 视图结构
建议由五个区块组成：

1. 顶部栏
2. Hero 区
3. 知识宇宙总览区
4. 继续探索区
5. 全部知识库卡片流

### 顶部栏
包含：
- 产品名 `Tree Knowledge`
- 全局搜索入口
- `新建知识库` 主按钮
- 设置入口
- Provider 入口

### Hero 区
作用是先建立产品气质，再分发用户进入具体知识库。

建议内容：
- 主标题：强调“从问题长成知识网络”
- 辅助说明：强调可回跳、可分叉、可延伸
- 大搜索框
- 主按钮 `开始一个新问题`
- 次按钮 `继续最近探索`
- 动态统计卡：
  - 知识库总数
  - 总节点数
  - 今日新增节点
  - 最近活跃知识库

### 知识宇宙总览区
这是启动台的主要视觉记忆点。

表现形式建议：
- 采用 `知识星图` 或 `知识群岛` 视图
- 每个知识库表现为一个知识团块
- 大小代表节点规模
- 发光或外圈扩散代表活跃度和最近增长
- 悬停显示名称、简介、根问题、节点数和最近探索节点
- 点击直接进入对应知识库工作台

### 继续探索区
作用是降低用户决策成本，让用户不用先思考“该进哪个库”。

建议分组：
- `最近访问`
- `最近增长`
- `等待继续`

### 全部知识库卡片流
这是稳定的信息落点，用于完整浏览和筛选。

每张卡片建议包含：
- 知识库名称
- 一句话简介
- 根问题
- 节点数量
- 最近更新时间
- 最近 3 个热点节点标签
- 迷你图谱预览
- 状态标签

### 空状态
当没有任何知识库时，不展示空白列表。

应直接展示：
- 一句引导文案
- 一个大输入框：`你现在最想探索什么？`
- 创建第一个知识库的主按钮

## 2. 创建/编辑知识库视图
### 目标
- 创建新的知识空间
- 定义知识库名称、简介和起始问题

### 核心模块
- 知识库名称
- 知识库简介
- 起始问题输入框
- 创建按钮

### 视图逻辑
- 用户输入名称与起始问题
- 系统创建知识库
- 系统自动生成根节点
- 创建成功后跳转到该知识库的工作台

## 3. 知识工作台
### 视图定位
知识工作台是系统的主操作视图。  
大部分知识操作都应在这里完成，而不是拆散到多个跳转视图中。

### 视图目标
- 围绕当前节点继续扩展知识
- 看清当前节点的位置和关系
- 在不同节点之间切换时保持连续性

### 视图结构
- 顶部栏
- 左栏导航
- 中间图谱画布
- 右栏详情区
- 底部输入区

### 顶部栏
包含：
- 返回桌面启动台
- 当前知识库名称
- 全局搜索入口
- 知识库设置入口
- Provider 快速状态入口

### 左栏
包含：
- 当前节点祖先路径
- 当前节点直接子节点
- 最近访问节点

### 中栏
包含：
- 当前节点为中心的图谱
- 祖先、子节点、相关节点的有限半径可视化

### 右栏
包含：
- 节点标题
- 摘要
- 正文
- 来源问答
- 相关节点
- AI 建议

### 底部输入区
支持：
- 直接生成节点
- 请求 AI 生成候选节点

### 视图逻辑
- 点击任意节点，切换当前节点，不离开工作台
- 当前节点切换后，左栏、中栏、右栏同步刷新
- 除知识库切换外，用户应尽量停留在此视图内完成探索

## 4. 候选节点确认层
### 形式
建议做成抽屉或弹层，不单独切换完整视图。

### 目标
- 承接 AI 给出的候选扩展主题
- 在落库前让用户确认结构和内容方向

## 5. 全局搜索层
### 形式
建议先做成全局浮层。

### 目标
- 在当前知识库内快速定位节点

## 6. 节点结构编辑层
### 形式
建议优先做成右栏编辑态，复杂操作使用辅助弹层。

### 目标
- 修改节点内容
- 修正节点的结构归属和关系边

## 7. 知识库设置视图
### 目标
- 管理知识库级别配置

## 8. Provider 管理视图
### 目标
- 管理用户自己的 LLM Provider
- 测试 Provider 连通性
- 绑定默认模型

### 模块
- Provider 列表
- 新增 Provider
- Base URL
- API Key
- Model 列表
- 测试连接
- 默认模型绑定

## 9. 图谱全览页
### 定位
后期增强视图，不作为 MVP 核心入口。

## 主流程
最重要的视图主流程如下：

1. 用户进入 `桌面启动台`
2. 用户创建新知识库，或进入已有知识库
3. 用户按需进入 `Provider 管理视图` 配置自己的模型
4. 用户进入 `知识工作台`
5. 用户在工作台发起提问
6. 系统直接生成节点，或打开 `候选节点确认层`
7. 用户确认后刷新工作台
8. 用户通过 `全局搜索层` 或节点点击切换当前节点
9. 用户需要跨主题切换时返回 `桌面启动台`

## 视图优先级
### P0
- `桌面启动台`
- `创建/编辑知识库视图`
- `知识工作台`
- `Provider 管理视图`
- `候选节点确认层`
- `全局搜索层`

### P1
- `节点结构编辑层`
- `知识库设置视图`

### P2
- `图谱全览页`
- `导入视图`

## 视图设计原则
- 桌面启动台要像“知识宇宙总览”，不能像普通列表页
- 工作台要像“知识操作台”，不能像聊天页
- 节点切换尽量在单视图内完成，不制造多余跳转
- 辅助功能优先使用抽屉、弹层、编辑态，而不是新增完整视图
