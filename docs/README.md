# Tree Knowledge Docs

## 项目定义
Tree Knowledge 是一个本地优先、节点驱动的知识探索系统。它的目标不是保存长聊天记录，也不是提供传统 Wiki 页面树，而是把用户与 LLM 的探索过程沉淀为可回跳、可分叉、可关联的知识网络。

系统中的主对象是 `节点`，不是 `消息`：
- 节点是知识资产
- 消息是生成节点的材料
- 树结构负责定位
- 关系边负责联想

## MVP 范围
MVP 聚焦单用户、本地优先、图谱优先的知识工作台，只解决以下问题：
- 让用户先从桌面启动台感知和进入不同知识空间
- 从一个问题出发生成知识根节点
- 围绕当前节点继续提问并生成新节点
- 在任意祖先节点回跳后展开另一条分支
- 用横向关系把不同分支重新连接
- 用节点摘要而不是长聊天上下文驱动后续生成
- 让用户接入自己的 LLM 提供商，而不是依赖平台托管模型

MVP 不包含：
- 多人协作
- 权限系统
- 评论
- 云同步前置依赖
- 复杂版本历史
- 自定义关系类型体系

## 术语表
- `节点`：系统中的知识最小资产，包含标题、摘要、正文和结构关系
- `当前节点`：工作台中心节点，也是默认的上下文起点
- `主父节点`：节点在层级树中的唯一上级，用于定位和回跳
- `关系边`：节点之间的横向语义连接，用于支持联想和图谱探索
- `候选节点`：LLM 为用户生成的可选扩展主题，用户确认后才落库
- `桌面启动台`：桌面应用的主入口视图，用于展示所有知识库、总览其生长状态并进入具体工作台
- `工作台`：主视图，包含树路径、图谱画布、节点详情和输入区
- `Provider`：用户自行配置的模型提供商，例如 OpenAI、OpenRouter、Anthropic、Gemini、Ollama、LM Studio

## 关键设计原则
- 节点优先，不以消息流作为主导航
- 用户在知识库之间切换时离开工作台，在节点之间切换时尽量不离开工作台
- 结构可修正，用户对节点归属和关系拥有最终决定权
- 图谱是主视图，但数据模型先以本地关系型数据库落地
- LLM 负责建议和生成，不负责决定全局结构真相
- 上下文围绕当前节点组装，不传整段历史对话
- 模型凭证和知识数据默认由用户本地持有

## 建议实现顺序
1. 先实现桌面启动台、根节点创建、当前节点切换和三栏工作台
2. 再实现 Provider 配置、候选节点生成、直接扩展和图谱最小闭环
3. 然后补齐结构调整、上下文快照、搜索定位和本地导入导出
