# MVP 工作台

## 目标
MVP 工作台要解决两个问题：
- 让用户围绕当前节点持续展开知识，而不是堆积聊天记录
- 让用户随时看见当前节点的结构位置和可继续探索的方向

因此工作台必须围绕 `当前节点` 设计，而不是围绕 `当前会话` 设计。

工作台并不是产品的首页。  
用户先从 `知识库首页` 进入具体知识库，再在工作台内完成节点级的高频操作。

## 页面边界
为了避免页面职责混乱，MVP 页面边界定义如下：
- `知识库首页` 负责展示所有知识库、表达增长状态和分发进入
- `知识工作台` 负责节点浏览、节点生成、节点编辑和节点跳转
- `候选节点确认层`、`全局搜索层`、`节点结构编辑层` 是工作台上的辅助层

关键原则：
- 用户在知识库之间切换时离开工作台
- 用户在节点之间切换时尽量不离开工作台

## 页面结构
主页面采用三栏结构，图谱画布位于中栏，是默认视觉中心。

### 左栏：结构导航
左栏负责回答“我从哪来”和“我还能快速跳去哪”。

必须包含：
- 当前节点的祖先路径
- 当前节点的直接子节点列表
- 最近访问节点

默认行为：
- 点击祖先路径中的任意节点，会把该节点切为当前节点
- 点击直接子节点，会进入该节点并刷新中栏图谱和右栏详情
- 最近访问节点按时间倒序，帮助用户在多个分支间切换

不做：
- 全量树展开
- 无限级折叠管理
- 复杂拖拽重排

### 中栏：图谱画布
中栏负责回答“当前节点和周围知识结构是什么关系”。

默认渲染范围：
- 当前节点
- 祖先链
- 当前节点的直接子节点
- 当前节点的相关节点

默认限制：
- 首版只渲染有限半径节点
- 相关节点数量应有上限，避免首屏过载
- 图谱布局只需可读，不需要复杂物理模拟

交互要求：
- 当前节点始终高亮
- 点击节点可切换当前节点
- 节点之间区分层级边和关系边
- 鼠标悬停可展示节点标题与关系类型

不做：
- 全局自由漫游图谱
- 大规模节点拖拽排布编辑
- 首版图算法优化

### 右栏：当前节点详情
右栏负责回答“这个节点到底是什么”和“下一步怎么处理它”。

必须包含：
- 节点标题
- 节点摘要
- 节点正文
- 来源信息
- 相关节点列表
- AI 建议列表

默认操作：
- 编辑标题
- 编辑摘要
- 编辑正文
- 添加关系边
- 查看来源问答
- 归档节点

### 底部输入区
输入区始终绑定当前节点，用于围绕它继续提问或扩展。

必须支持两种模式：
- 直接生成节点
- 生成候选节点

#### 直接生成节点
适用于用户意图明确的情况，例如：
- “解释这个概念的底层机制”
- “继续拆成三个步骤”
- “给我一个反例”

系统返回一个新节点草稿，并给出默认落位建议。

#### 生成候选节点
适用于探索式意图，例如：
- “还能从哪些方向继续看”
- “这个主题的关键分支有哪些”
- “我应该下一步研究什么”

系统返回 3 到 5 个候选扩展主题，用户选择后再创建节点。

## 节点创建流程
### 根节点创建
1. 用户输入第一个问题
2. 系统调用 LLM 生成根节点标题、摘要、正文
3. 系统创建根节点并将其设为当前节点
4. 系统从知识库首页跳转进入该知识库工作台

### 从当前节点扩展
1. 用户在输入区发起问题或请求建议
2. 系统读取当前节点及必要上下文
3. LLM 返回候选节点或直接生成结果
4. 系统建议落位方式：
   - `child`
   - `branch`
   - `related`
5. 用户确认后正式创建节点
6. 工作台刷新结构导航、图谱和详情

### 结构修正
用户必须能在创建后修正结构：
- 修改节点标题和摘要
- 调整主父节点
- 调整关系边
- 归档无效节点

## 搜索与定位
MVP 只提供全局搜索，不额外单独设计复杂检索页。

搜索结果卡片必须包含：
- 节点标题
- 节点摘要
- 所在路径
- 关系数量或标签

点击结果后的默认行为：
- 将结果节点切换为当前节点
- 重新构建图谱画布
- 在左栏显示它的祖先路径和子节点

## 节点管理范围
MVP 内必须支持：
- 重命名
- 编辑摘要和正文
- 查看来源问答
- 调整主父节点
- 新增和删除关系边
- 节点归档

MVP 不支持：
- 节点删除恢复站
- 合并多个节点
- 节点版本比较
- 批量操作

## 默认交互规则
- 任何输入默认以当前节点为上下文起点
- 如果用户没有指定模式，系统根据提问意图判断是直接生成还是候选生成
- 如果系统无法判断节点应落在哪，默认创建为当前节点的子节点
- 节点创建后立即可见于图谱和左栏导航
- 未确认的候选节点不得进入正式图谱
- 返回知识库级入口时，应回到知识库首页，而不是其他节点页

## 验收场景
- 从根节点连续扩展至少 20 个节点，工作台仍能稳定导航
- 在第 4 层节点回跳到第 2 层后创建新分支，原分支结构不受污染
- 从候选列表中只选择 1 个主题生成，其他候选不会进入正式数据
- 搜索任意节点后可恢复其周边结构和当前上下文
- 用户修改父节点后，图谱和左栏路径同步更新
