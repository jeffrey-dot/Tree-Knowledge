# MVP 工作台

## 目标
MVP 工作台要解决两个问题：
- 让用户围绕当前节点持续展开知识，而不是堆积聊天记录
- 让用户随时看见当前节点的结构位置和可继续探索的方向

因此工作台必须围绕 `当前节点` 设计，而不是围绕 `当前会话` 设计。

工作台并不是产品的启动入口。  
用户先从 `桌面启动台` 进入具体知识库，再在工作台内完成节点级的高频操作。

## 视图边界
为了避免视图职责混乱，MVP 视图边界定义如下：
- `桌面启动台` 负责展示所有知识库、表达增长状态和分发进入
- `知识工作台` 负责节点浏览、节点生成、节点编辑和节点跳转
- `候选节点确认层`、`全局搜索层`、`节点结构编辑层` 是工作台上的辅助层
- `Provider 管理视图` 负责配置用户自己的模型提供商和默认模型映射

关键原则：
- 用户在知识库之间切换时离开工作台
- 用户在节点之间切换时尽量不离开工作台

## 视图结构
主视图采用三栏结构，图谱画布位于中栏，是默认视觉中心。

### 左栏：结构导航
必须包含：
- 当前节点的祖先路径
- 当前节点的直接子节点列表
- 最近访问节点

### 中栏：图谱画布
默认渲染范围：
- 当前节点
- 祖先链
- 当前节点的直接子节点
- 当前节点的相关节点

### 右栏：当前节点详情
必须包含：
- 节点标题
- 节点摘要
- 节点正文
- 来源信息
- 相关节点列表
- AI 建议列表

### 底部输入区
必须支持两种模式：
- 直接生成节点
- 生成候选节点

## 节点创建流程
### 根节点创建
1. 用户在桌面启动台创建知识库
2. 系统调用 LLM 生成根节点标题、摘要、正文
3. 系统创建根节点并将其设为当前节点
4. 系统从桌面启动台跳转进入该知识库工作台

### 从当前节点扩展
1. 用户在输入区发起问题或请求建议
2. 系统读取当前节点及必要上下文
3. LLM 返回候选节点或直接生成结果
4. 系统建议落位方式：`child`、`branch`、`related`
5. 用户确认后正式创建节点
6. 工作台刷新结构导航、图谱和详情

## Provider 依赖
工作台需要感知当前可用的 Provider 状态：
- 未配置：输入区可见但提交前提醒用户先配置 Provider
- 已配置：正常生成
- 当前模型不可用：显示轻量错误状态并允许切换 Provider

## 默认交互规则
- 任何输入默认以当前节点为上下文起点
- 如果用户没有指定模式，系统根据提问意图判断是直接生成还是候选生成
- 如果系统无法判断节点应落在哪，默认创建为当前节点的子节点
- 节点创建后立即可见于图谱和左栏导航
- 未确认的候选节点不得进入正式图谱
- 返回知识库级入口时，应回到桌面启动台，而不是其他节点视图
