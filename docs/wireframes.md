# 线框草图说明

## 目标
这份文档定义桌面启动台与工作台的低保真线框方向，核心原则只有三条：

- 界面简洁
- 文案极少
- 优先用图标、布局和状态变化引导用户

## 全局界面原则
- 页面上的长文本只出现在节点正文区，不应出现在导航和控制区
- 图标旁仅在首次使用、hover 或选中态时显示短标签
- 默认高亮主操作按钮，不做重文案 onboarding

## 1. 桌面启动台线框
### 视图目标
用户进入应用后，不应该看到普通列表或介绍页，而应该像走近一面贴满便利贴的知识墙，先看见知识块，再决定进入哪里。

### 桌面端线框结构
```text
+----------------------------------------------------------------------------------+
| Logo                         Search                  + New   Providers  Settings |
+----------------------------------------------------------------------------------+
|   [ + Start ] [ ↺ Continue ]                         [ latest workspace ]        |
+----------------------------------------------------------------------------------+
| Knowledge Wall                                                                   |
|                                                                                  |
|   [ sticky note ]   [ sticky note ]   [ sticky note ]                           |
|   [ sticky note ]   [ sticky note ]   [ sticky note ]                           |
|                                                                                  |
+----------------------------------------------------------------------------------+
| Providers: [ default ] [ status ] [ ⇢ ]                                         |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

## 2. 知识工作台线框
### 视图目标
工作台必须把用户注意力集中在 `当前节点` 与 `周边结构`，不是菜单、说明或消息流。

### 桌面端线框结构
```text
+----------------------------------------------------------------------------------+
| [⌂] Library Name               [provider]                                [⌕] [⚙] |
+-------------------+------------------------------------------------+-------------+
| path              |                                                | note title   |
|  o root           |                                                | summary      |
|  o branch         |                GRAPH CANVAS                    | body         |
|  ◎ current        |                                                | source       |
|                   |                                                | relations    |
| children          |                                                | suggestions  |
|  o child 1        |                                                | [ edit ]     |
|  o child 2        |                                                |              |
|                   |                                                |              |
| recent            |                                                |              |
|  o node a         |                                                |              |
|  o node b         |                                                |              |
+-------------------+------------------------------------------------+-------------+
| [ child ] [ branch ] [ related ] [ input .................... ] [候选] [长出]    |
+----------------------------------------------------------------------------------+
```

## 3. 规则摘要
- 启动台不堆很多管理按钮
- 工作台顶部不做多级菜单
- 搜索、Provider、设置优先图标化
- 编辑优先原地展开，不开新视图
- 知识库卡片应当更像便利贴，而不是后台卡片
- 右栏先读后改，默认不展示大面积表单
- 入口区不写理念说明，只保留动作
