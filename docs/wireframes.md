# 线框草图说明

## 目标
这份文档定义首页与工作台的低保真线框方向，核心原则只有三条：

- 界面简洁
- 文案极少
- 优先用图标、布局和状态变化引导用户

这不是视觉稿，而是给设计和前端对齐页面骨架、信息密度和交互提示方式的说明。

## 全局界面原则
### 1. 少字原则
- 不做大段解释文案
- 非必要不写说明句
- 默认通过位置、图标、层级和 hover 提示表达含义
- 页面上的长文本只出现在节点正文区，不应出现在导航和控制区

### 2. 图标优先原则
- 高频操作优先使用图标按钮
- 图标旁仅在首次使用、hover 或选中态时显示短标签
- 同类动作保持固定图标语义，不频繁换图形

建议图标语义：
- `+`：新建
- `⌕`：搜索
- `◎`：当前节点 / 当前知识库
- `↺`：继续探索 / 回到最近
- `⇢`：进入 / 打开
- `⋯`：更多操作
- `⌂`：返回首页
- `⚙`：设置
- `⤴`：切换分支 / 跳转
- `⊕`：添加关联
- `✎`：编辑
- `✦`：AI 建议 / 智能扩展

最终实现时可替换成统一图标库，但语义必须稳定。

### 3. 引导方式
MVP 不依赖长文字 onboarding，而采用以下方式引导：
- 默认高亮主操作按钮
- hover 时出现极短提示
- 首次使用时只展示一次轻提示层
- 用空状态构造第一步动作
- 用动态反馈告诉用户“点这里会发生什么”

### 4. 信息密度原则
- 首页优先看“知识库状态”，不是读说明
- 工作台优先看“结构”和“当前节点”，不是读菜单
- 控制按钮集中放在边缘，中心区域留给内容和图谱

## 1. 知识库首页线框
### 页面目标
用户进入首页后，不应该看到普通列表，而应该先看到一个“知识宇宙”的总览感，再自然进入某个知识库。

### 桌面端线框结构
```text
+----------------------------------------------------------------------------------+
| Logo                         Search                        + New        Settings |
+----------------------------------------------------------------------------------+
|                                                                                  |
|   [ Hero title / minimal line ]                  [ abstract network visual ]      |
|   [ large start input ...................................................... ]    |
|   [ + ]  [ ↺ ]                                                                  |
|                                                                                  |
|   [ stat ] [ stat ] [ stat ] [ stat ]                                           |
|                                                                                  |
+----------------------------------------------------------------------------------+
|                             Knowledge Universe Map                               |
|                                                                                  |
|           o                OOO                       oo                           |
|                 ooo                 O                     o                      |
|      oo                    oooo                oo                                |
|                                                                                  |
+----------------------------------------------------------------------------------+
| Continue:   [ card ] [ card ] [ card ]                                          |
+----------------------------------------------------------------------------------+
| Filters: [●] [●] [●] [●]       View: [Map/Grid]         Sort: [icon]            |
|                                                                                  |
|   [ library card ]   [ library card ]   [ library card ]                        |
|   [ library card ]   [ library card ]   [ library card ]                        |
|                                                                                  |
+----------------------------------------------------------------------------------+
```

## 1.1 顶部导航线框说明
### 结构
- 左：Logo
- 中：全局搜索入口
- 右：新建、设置、用户入口

### 规则
- 顶栏只保留图标和必要品牌名
- 搜索优先用搜索图标 + 可展开输入框
- 新建知识库使用最醒目的圆角图标按钮
- 设置和用户入口不使用文字标签

### 行为
- hover 才显示简短 tooltip
- 默认不显示“新建知识库”“设置”等长文字

## 1.2 Hero 区线框说明
### 结构
- 左侧一条极短主句
- 下方一个大输入框，作为主要入口
- 输入框下只保留两个图标按钮：
  - `+`：创建新的知识起点
  - `↺`：回到最近探索
- 右侧是抽象知识网络背景，不放说明文字

### 规则
- Hero 区的文字压缩到最少
- 主标题建议控制在 8 到 14 个字
- 副标题可选，最多一行
- 统计区只展示数字和小标签，不写解释段落

## 1.3 知识宇宙总览区线框说明
### 结构
- 区块标题可弱化，甚至只保留一个小图标和短标题
- 中央是大面积总览图
- hover 某个知识库时，在附近出现信息浮卡

### 总览图表达
- 每个知识库是一个团块
- 团块大小表示规模
- 团块亮度表示活跃度
- 外圈扩散表示最近增长

### 交互
- 默认不展示文字标签铺满画面
- 只有 hover 或选中时，才显示：
  - 名称
  - 节点数
  - 最近更新时间

## 1.4 Continue 区线框说明
### 结构
- 不做复杂文案分组标题
- 每个分组标题用短词或图标：
  - `↺` 最近
  - `✦` 增长
  - `…` 待继续

### 卡片形式
```text
+----------------------+
| [icon] Library Name  |
| current node title   |
|     [ ⇢ ]            |
+----------------------+
```

### 规则
- 每张卡仅保留一行主标题和一行次级信息
- 主操作用箭头图标，不放长按钮文案

## 1.5 全部知识库卡片流线框说明
### 卡片线框
```text
+------------------------------+
| [mini graph]            [⋯] |
| Library Name                 |
| short topic                  |
| [tag] [tag] [tag]            |
|      [ ⇢ ]                   |
+------------------------------+
```

### 规则
- 不展示冗长描述
- 不直接显示完整根问题全文
- 以短标题、标签、迷你图谱和状态点表达信息
- 更多操作收进 `⋯`

### 筛选条
- 使用图标切换，不使用长下拉描述
- 选中项用高亮点或底部短线表示

## 1.6 首页空状态线框
```text
+---------------------------------------------------------------+
|                                                               |
|                         [ abstract node ]                     |
|                                                               |
|               [ start input ................................ ]|
|                           [ + ]                               |
|                                                               |
+---------------------------------------------------------------+
```

### 规则
- 空状态用一个中心输入框解决，不做说明页
- 用户第一次进入时，应直接知道“从这里开始”

## 2. 知识工作台线框
### 页面目标
工作台必须把用户注意力集中在 `当前节点` 与 `周边结构`，不是菜单、说明或消息流。

### 桌面端线框结构
```text
+----------------------------------------------------------------------------------+
| [⌂]   Library Name                             [⌕] [✦] [⚙]                       |
+-------------------+------------------------------------------------+-------------+
| ancestors         |                                                | title   [✎] |
|  o root           |                                                | summary      |
|  o branch         |                GRAPH CANVAS                    | body         |
|  ◎ current        |                                                | relations    |
|                   |                                                | source       |
| children          |                                                | suggestions  |
|  o child 1        |                                                |              |
|  o child 2        |                                                |              |
|                   |                                                |              |
| recent            |                                                |              |
|  o node a         |                                                |              |
|  o node b         |                                                |              |
+-------------------+------------------------------------------------+-------------+
| [mode icon]  [ (Ref Node A) (Ref Node B) x ] [ input ... ]               [ ⇢ ]  |
+----------------------------------------------------------------------------------+
```

## 2.1 顶部栏线框说明
### 结构
- 左：返回首页图标
- 中：当前知识库名称
- 右：搜索、AI、设置

### 规则
- 不做文字菜单导航
- 顶栏所有动作优先图标化
- 当前知识库名称是唯一长期可见文字

## 2.2 左栏线框说明
### 结构
- 祖先链
- 子节点列表
- 最近访问

### 表达方式
- 节点使用小圆点或小图标 + 极短标题
- 当前节点用 `◎` 高亮
- 不显示长摘要

### 规则
- 左栏是跳转层，不是阅读层
- 每个节点只占一行
- 超长标题截断

## 2.3 中央图谱线框说明
### 结构
- 左上角工具按钮组
- 中央图谱
- hover 时出现小浮层

### 工具按钮
- `＋` / `－`：缩放
- `⊙`：重置视角
- `⇄`：切换关系显示

### 图谱节点
- 当前节点最大
- 祖先节点小而稳定
- 子节点分布在当前节点外圈
- 相关节点使用不同线型或不同节点轮廓

### 规则
- 默认不在画布上铺满名称
- 只显示当前节点名称和必要邻近节点
- hover 才显示更多信息

## 2.4 右栏节点详情线框说明
### 结构
- 顶部标题行
- 摘要块
- 正文块
- 关系块
- 来源块
- AI 建议块

### 压字原则
- 标题区保留标题
- 摘要区尽量控制在 2 到 4 行
- 正文默认折叠
- 来源信息默认只显示图标入口

### 图标引导
- `✎`：编辑
- `⊕`：添加关系
- `✦`：生成建议
- `…`：更多节点操作

### 规则
- 右栏是“当前节点信息面板”，不是文档阅读页
- 长文默认折叠，以减少压迫感

## 2.5 底部输入区线框说明
### 结构
- 左：当前模式图标
- 中：主输入框
- 右：提交图标按钮

### 模式图标
- `✦`：候选模式
- `⇢`：直接生成

### 规则
- 不用大段 placeholder 教用户
- placeholder 只保留极短引导，例如：`继续问`
- 当前上下文提示用小胶囊展示当前节点标题

## 3. 辅助层线框
### 3.1 候选节点确认层
```text
+-------------------------------+
| [✦]                           |
| candidate a              [⇢]  |
| candidate b              [⇢]  |
| candidate c              [⇢]  |
|                               |
| selected summary              |
|                     [ + ]     |
+-------------------------------+
```

规则：
- 默认先展示候选列表
- 文案压缩为标题 + 一小段摘要
- 确认按钮使用图标主按钮

### 3.2 搜索层
```text
+--------------------------------------------------+
| [⌕ ................................. ]           |
|  result title                             [⇢]    |
|  result title                             [⇢]    |
|  result title                             [⇢]    |
+--------------------------------------------------+
```

规则：
- 搜索结果一行一个
- 不展示长摘要，只展示短路径和标题

### 3.3 编辑态
规则：
- 编辑优先原地展开
- 不开新页面
- 输入区域和阅读区域尽量共用同一布局

## 4. 首次使用引导
### 原则
- 只做一次性轻提示
- 不做多步教程弹窗
- 不遮挡主界面

### 首页引导
- 高亮 Hero 输入框
- 高亮 `+`

### 工作台引导
- 高亮当前节点
- 高亮底部输入框
- 高亮图谱中的第一个可点击子节点

## 5. 移动端简化原则
- 首页保留 Hero、总览缩略图和卡片流
- 工作台收起左栏，改为顶部节点抽屉
- 右栏详情改为底部抽屉
- 图谱画布保留最小浏览能力，不做复杂编辑

## 6. 禁止项
- 用大量说明文字解释如何使用
- 在首页堆很多管理操作按钮
- 在工作台顶部放多级菜单
- 在图谱里显示过多节点文字
- 把输入区做成传统聊天气泡流
