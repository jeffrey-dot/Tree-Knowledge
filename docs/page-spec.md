# 页面细化规格

## 目标
这份文档用于把 `页面地图` 继续收敛成可以直接进入线框设计和前端实现的页面规格。  
重点覆盖：
- 首页和工作台的模块层级
- 页面状态
- 关键交互
- 桌面端优先布局

MVP 以桌面端体验为主，移动端优先保证可访问和基础操作，不追求完整画布体验。

## 1. 知识库首页规格
### 页面定位
知识库首页是 `知识宇宙总览页`，不是普通列表页。  
它需要同时承担品牌首页、知识库分发页和知识状态总览页三种职责。

### 桌面端页面结构
建议按以下顺序布局：

1. 顶部导航
2. Hero 区
3. 知识宇宙总览区
4. 继续探索区
5. 全部知识库卡片流
6. 归档或次级入口区

### 1.1 顶部导航
#### 模块
- Logo / 产品名
- 全局搜索入口
- 新建知识库按钮
- 设置入口
- 用户入口

#### 布局
- 左：Logo / 产品名
- 中：搜索入口
- 右：新建按钮、设置、用户入口

#### 交互
- 点击 Logo 回到首页顶部
- 点击搜索打开全局搜索层
- 点击新建知识库打开创建页或创建弹层

#### 状态
- 默认态
- 滚动吸顶态

### 1.2 Hero 区
#### 模块
- 主标题
- 副标题
- 主搜索框
- 主按钮 `开始一个新问题`
- 次按钮 `继续最近探索`
- 统计卡组

#### 推荐布局
- 左侧为文案和输入
- 右侧为轻视觉背景或抽象知识网络

#### 统计卡组字段
- 知识库总数
- 总节点数
- 今日新增节点
- 最近活跃知识库

#### 交互
- 在主搜索框输入问题并提交，直接进入创建知识库流程
- 点击 `继续最近探索` 进入最近活跃知识库

#### 状态
- 首次使用态：强化“创建第一个知识库”
- 常规使用态：强化“继续最近探索”

### 1.3 知识宇宙总览区
#### 模块
- 区块标题
- 总览图画布
- 悬浮信息卡

#### 视觉要求
- 每个知识库以星团、岛屿或团块方式呈现
- 团块大小表达节点规模
- 边缘扩散或亮度表达活跃度
- 不追求严格图算法，但要有明确主次层次

#### 悬浮信息卡字段
- 知识库名称
- 一句话简介
- 根问题
- 节点数量
- 最近更新时间
- 最近探索节点

#### 交互
- hover：高亮该知识库并显示悬浮信息卡
- click：进入对应工作台

#### 状态
- 正常态：展示全部活跃知识库
- 空态：隐藏总览图，回退为首页空状态引导

### 1.4 继续探索区
#### 模块
- 分组标题
- 横向卡片组

#### 分组建议
- 最近访问
- 最近增长
- 等待继续

#### 卡片字段
- 知识库名称
- 推荐继续的节点标题
- 推荐原因
- 最后更新时间
- `继续探索` 按钮

#### 逻辑
- 用户优先从这里重新进入已有知识库
- 推荐原因由规则计算生成，不需要 MVP 中由 LLM 生成

### 1.5 全部知识库卡片流
#### 模块
- 视图切换器（星图 / 网格视图切换）
- 筛选条
- 排序条
- 卡片网格

#### 交互增强
- **磁吸搜索**：在“知识宇宙”模式下，搜索框输入时，匹配的星团（知识库）将自动高亮并平滑缩放至视口中心。
- **视图持久化**：系统应记住用户对首页视图（地图 vs 列表）的偏好。

#### 筛选项
- 最近活跃
- 节点最多
- 最近创建
- 待继续
- 已沉淀

#### 卡片字段
- 知识库名称
- 简介
- 根问题
- 节点数
- 最近更新时间
- 最近 3 个热点节点标签
- 迷你图谱预览
- 状态标签
- 主按钮 `继续探索`
- 更多菜单

#### 更多菜单
- 重命名
- 设置
- 归档
- 删除

#### 布局
- 桌面端采用 2 到 4 列响应式网格
- 卡片高度尽量一致，避免瀑布流破坏可比性

### 1.6 首页空状态
#### 模块
- 主标题
- 解释文案
- 大输入框
- 创建按钮

#### 默认文案方向
- `从第一个问题开始，建立你的知识网络`
- 输入框占据视觉中心

#### 交互
- 输入问题后直接进入创建知识库流程

## 2. 知识工作台规格
### 页面定位
知识工作台是系统的主操作面。  
所有高频节点操作默认都应在这里完成，不再拆出额外页面。

### 桌面端页面结构
建议采用固定五区结构：

1. 顶部栏
2. 左栏结构导航
3. 中央图谱画布
4. 右栏节点详情
5. 底部输入区

### 2.1 顶部栏
#### 模块
- 返回首页按钮
- 当前知识库名称
- 面包屑或当前视图标签
- 搜索入口
- 知识库设置入口
- 当前节点快捷操作

#### 交互
- 点击返回首页，回到知识库首页
- 点击知识库名称，可打开知识库摘要卡
- 点击搜索，打开全局搜索层

### 2.2 左栏结构导航
#### 模块定位
左栏定位为“全局静态路径与书签”，主要负责在大跨度节点切换时提供坐标参考。
- 当前节点祖先全路径
- 关键节点书签（用户标记）
- 历史回溯

#### 交互
- **响应式折叠**：允许用户一键折叠左栏，为中栏图谱腾出更多横向探索空间。
- **层级跳转**：点击祖先节点，工作台中心立即切换至该节点。
- **书签快捷键**：支持通过快捷键或拖拽节点至左栏生成书签。

#### 状态
- 默认态
- 当前节点无子节点态
- 当前节点为根节点态

### 2.3 中央图谱画布
#### 模块
- 画布工具栏
- 图谱画布
- 节点悬浮提示

#### 工具栏建议
- 视图重置
- 缩放
- 只看层级
- 显示关系边

#### 画布内容
- 当前节点
- 祖先链
- 直接子节点
- 相关节点

#### 交互
- 点击节点：切换当前节点
- hover 节点：显示标题、摘要、关系类型
- 缩放和平移：只影响当前画布视角，不改变结构数据

#### 状态
- 正常态
- 节点较少态
- 加载态

### 2.4 右栏节点详情
#### 模块
- 标题区
- 摘要区
- 正文区
- 来源信息区
- 相关节点区
- AI 建议区

#### 标题区
包含：
- 节点标题
- 状态标签
- 编辑按钮

#### 摘要区
包含：
- 摘要内容
- 重新生成摘要入口（后期）

#### 正文区
包含：
- 正文内容
- 展开 / 收起

#### 来源信息区
包含：
- 来源提问
- 来源回答摘要
- 查看完整来源入口

#### 相关节点区
包含：
- 关系列表
- 添加关系边按钮

#### AI 建议区
包含：
- 3 到 5 条下一步建议
- 一键进入候选生成

### 2.5 底部输入区
#### 模块
- 输入框
- **上下文气泡栈 (Context Bubble Stack)**
- 模式切换（直接生成 / 候选建议）
- 提交按钮

#### 交互
- **上下文气泡栈**：输入框上方以标签形式展示当前影响 AI 生成的上下文节点。用户可点击标签上的“x”移除特定的背景参考，或通过“+”手动添加非当前路径的节点作为背景。
- **快速分支 (Fast Branching)**：使用 `Shift + Enter` 提交时，AI 直接生成排名前 3 的逻辑分支并自动落位为子节点，无需通过“候选确认层”。
- **渐进式占位符**：根据当前节点状态，显示不同的引导文字，如“基于此节点继续深挖...”或“尝试换个方向联想...”。

## 3. 辅助层规格
### 3.1 候选节点确认层
#### 形式
- 右侧抽屉优先

#### 模块
- 当前问题摘要
- 候选列表
- 候选详情
- 确认创建
- 重新生成

#### 交互
- 选中某个候选后显示更详细摘要
- 确认后关闭抽屉并刷新工作台

### 3.2 全局搜索层
#### 形式
- 居中浮层

#### 模块
- 搜索输入框
- 搜索结果列表
- 快捷提示

#### 结果字段
- 节点标题
- 节点摘要
- 所在路径
- 所属知识库

#### 交互
- 在首页可搜索知识库和节点
- 在工作台内优先搜索当前知识库节点

### 3.3 节点结构编辑层
#### 形式
- 右栏编辑态 + 辅助弹层

#### 模块
- 标题编辑
- 摘要编辑
- 正文编辑
- 父节点调整
- 关系边编辑
- 归档确认

## 4. 页面状态清单
### 知识库首页
- 空状态
- 初次使用状态
- 常规状态
- 搜索筛选状态

### 知识工作台
- 根节点状态
- 普通节点状态
- 无子节点状态
- 候选生成中状态
- 节点编辑状态

## 5. 桌面端尺寸建议
### 首页
- 顶部导航：64px 到 72px
- Hero：首屏占据 70vh 左右
- 卡片流最大内容宽度：1200px 到 1440px

### 工作台
- 顶部栏：56px 到 64px
- 左栏：280px 到 320px
- 中栏：自适应主区域
- 右栏：360px 到 420px
- 底部输入区：固定悬浮或吸底

## 6. MVP 不做的视觉和交互
- 首页总览区自由拖拽排布
- 工作台中的复杂协同编辑
- 多窗口图谱比较
- 移动端完整图谱操作
- 首页和工作台中高成本 3D 动效

## 7. 实现优先级
### P0
- 首页 Hero
- 首页知识宇宙总览区
- 首页卡片流
- 工作台五区结构
- 候选节点确认层

### P1
- 首页继续探索区
- 工作台工具栏
- 节点结构编辑层

### P2
- 更丰富的首页状态标签
- 更复杂的迷你图谱预览
- 图谱全览页
