# 高保真视觉规范

## 目标
这份文档把线框草图推进为可执行的高保真视觉方向，目标不是做一套通用 SaaS 后台，而是建立一个 `知识操作系统` 风格的界面语言。

核心要求：
- 简洁
- 少字
- 图标优先
- 有记忆点
- 不落入常见 AI 产品的默认审美

## 视觉定位
### 产品气质
整体气质应接近：
- 研究工作台
- 知识操作系统
- 安静但有生命力的探索界面

不要接近：
- 普通管理后台
- Notion 风白底卡片堆
- 聊天机器人产品
- 炫技型科幻大屏

### 视觉关键词
- calm
- precise
- spatial
- luminous
- restrained

### 总体方向
- 以浅底为主，不默认深色
- 用微光、模糊、细线和层次阴影表达“网络生长”
- 用块面留白表达秩序
- 用局部发光表达活跃和可操作对象

## 颜色系统
### 基础色板
建议采用偏冷的中性色为底，配一组有节制的矿物色点亮。

#### 背景色
- `bg.base`: `#F5F3EE`
- `bg.surface`: `#FBF9F4`
- `bg.panel`: `rgba(255, 252, 245, 0.78)`
- `bg.elevated`: `#FFFDF8`

#### 文字色
- `text.primary`: `#1D1F1A`（确保对比度符合 WCAG 2.1 AAA 级标准）
- `text.secondary`: `#4E514B`（对比度提升，确保在纸质背景下清晰可见）
- `text.muted`: `#787D74`（最低限度的可读色）

#### 线条色
- `line.soft`: `rgba(51, 61, 44, 0.08)`
- `line.normal`: `rgba(51, 61, 44, 0.14)`
- `line.strong`: `rgba(51, 61, 44, 0.22)`

### 功能强调色
只保留少量角色明确的强调色。

- `accent.primary`: `#244A3F`
- `accent.primary.soft`: `#DCE9E1`
- `accent.gold`: `#C59B52`
- `accent.blue`: `#4E6F8C`
- `accent.coral`: `#C86B52`

### 状态色映射
- 当前节点：`accent.primary`
- AI 建议：`accent.gold`
- 关系边高亮：`accent.blue`
- 待处理 / 待继续：`accent.coral`

### 使用规则
- 启动台不超过 3 个主色同时竞争
- 单个模块只允许 1 个主强调色
- 活跃态用亮度和边框，不靠高饱和大片铺色

## 背景与氛围
### 启动台背景
- 使用大面积柔和径向渐变
- 叠加极轻的节点连线纹理
- 禁止复杂插画和具象图片

建议背景组合：
- 左上暖白光斑
- 右侧冷绿色雾化渐变
- 中间极轻的网络点阵

### 工作台背景
- 采用更克制的平静背景
- 图谱区域允许有淡淡网格或柔和光晕
- 侧栏和右栏使用半透明面板提升层次

## 字体系统
### 原则
不要使用默认系统后台组合。  
字体需要同时传达“结构感”和“阅读稳定性”。

### 推荐组合
- 标题字体：`Manrope` 或 `Sora`
- 正文字体：`IBM Plex Sans` 或 `Source Sans 3`
- 数字和标签：沿用标题字体

### 层级建议
- Hero 主标题：`40-56px`，中高字重
- 页面区块标题：`18-24px`
- 卡片标题：`16-18px`
- 正文摘要：`14-15px`
- 微标签：`11-12px`

### 用法规则
- 标题尽量短，允许更大字号
- 辅助说明严格限制行数
- 标签与数字要精确利落，不做松散排版

## 圆角、阴影与边框
### 圆角
- 主卡片：`24px`
- 二级面板：`20px`
- 输入框：`18px`
- 胶囊标签：`999px`
- 图标按钮：`14px` 到 `18px`

### 阴影
采用低对比、层叠式阴影，不要厚重悬浮。

建议：
- `shadow.soft`: `0 12px 30px rgba(35, 42, 30, 0.06)`
- `shadow.mid`: `0 18px 48px rgba(35, 42, 30, 0.08)`
- `shadow.glow`: `0 0 0 1px rgba(197, 155, 82, 0.08), 0 10px 30px rgba(197, 155, 82, 0.12)`

### 边框
- 默认细边框
- 强调态通过边框亮度变化表达
- 避免粗描边和纯黑分割线

## 图标系统
### 风格
- 使用线性图标为主
- 统一 1.75px 到 2px stroke
- 圆角端点
- 不使用过度拟物或填充型图标

### 行为规则
- 高频操作固定图标语义
- 图标按钮必须有 hover 态和 tooltip
- 图标按钮之间的视觉权重必须分级

### 按钮等级
- 一级：带背景的主图标按钮
- 二级：轻底色图标按钮
- 三级：无底色纯图标

## 组件视觉规范
## 1. 顶部导航
### 启动台导航
- 半透明浅色浮层
- 轻模糊背景
- 左右两侧分组清晰
- 新建按钮使用主强调色底

### 工作台导航
- 比启动台更克制
- 以图标为主
- 当前知识库名称保持稳定居左

## 2. 输入框
### 启动台主输入框
- 高度 `56-64px`
- 大圆角
- 浅色玻璃面板效果
- 聚焦时边框轻发光

### 工作台底部输入框
- 吸底悬浮
- 左侧模式图标胶囊
- 中部输入
- 右侧单独提交按钮

输入框状态：
- 默认
- hover
- focus glow
- loading

## 3. 卡片
### 启动台知识库卡片
卡片结构：
- 顶部迷你图谱
- 标题
- 短副标题
- 标签组
- 主动作箭头

视觉规则：
- 卡片留白偏大
- 顶部迷你图谱先吸引眼睛
- 主动作按钮缩到右下或底部中心
- hover 时卡片轻微上浮，迷你图谱亮起

### Continue 卡片
- 尺寸更小
- 更强聚焦“继续进入”
- 操作按钮权重要高于信息量

## 4. 知识宇宙总览区
### 视觉表现
- 启动台核心视觉区应当像“漂浮的知识群落”
- 每个知识库使用发光团块 + 周边粒子
- 连接线极弱，只在 hover 时增强

### 动态要求
- 默认有缓慢呼吸感
- hover 某团块时，邻近粒子轻微联动
- 点击时有收束到目标知识库的过渡感

### 禁止项
- 不做游戏式星空
- 不做真实宇宙照片背景
- 不做五颜六色泡泡图

### 5. 工作台图谱
#### 画布风格
- 背景保持安静
- **层级约束布局**：为了避免节点密集的“毛线球”效应，中栏图谱应默认采用“层级向心”布局，即祖先节点分布在外圈或上方，当前节点占据视觉焦点，子节点呈放射状分布。

#### 节点样式
- **渐进式标签 (Progressive Labels)**：节点默认仅显示核心图标。当鼠标悬停或视图放大到一定比例时，平滑淡入完整标题。
- 当前节点：最大、最亮、主强调色描边。
- 祖先节点：小、稳定、低对比。
- 子节点：中小型，可点击高亮。
- 相关节点：边框虚实区别于层级节点。

#### 边样式
- 层级边：细实线。
- 关系边（横向关联）：更细的虚线，可带有流动的微光粒子，暗示联想的活跃性。
- hover 关系：线条加强并显示小标签。

### 标签显示
- 默认只显示当前节点和少量邻近节点名称
- 其他文字通过 hover 浮层展示

## 6. 右侧详情栏
### 风格
- 作为阅读面板，应更安静
- 采用浅面板 + 极轻阴影
- 模块之间用留白分隔，而不是重线框

### 布局
- 标题最大
- 摘要紧随其后
- 正文默认折叠成短段
- 来源和关系采用图标标题行

### 文字控制
- 摘要限制为 2 到 4 行
- 正文默认展开一屏以内
- 二级信息使用更浅色

## 7. Tooltip 与轻提示
### Tooltip
- 深浅反差适中
- 只写 1 到 4 个字或极短动作
- 出现速度快，消失柔和

### 首次提示
- 使用细边框高亮 + 小标签
- 不做厚重蒙层
- 不超过 3 个提示点

## 动效系统
### 原则
- 少而准
- 动效服务理解，不服务炫技

### 启动台动效
- Hero 背景缓慢漂移
- 知识团块轻微呼吸
- 卡片 hover 上浮 `4-6px`
- 筛选切换带短暂淡入淡出

### 工作台动效
- 当前节点切换时，中栏轻过渡重排
- 右栏内容切换用淡入和微位移
- 输入提交后出现短暂 loading pulse

### 时长建议
- 快速交互：`120-180ms`
- 面板切换：`180-240ms`
- 图谱重排：`240-360ms`

### 缓动建议
- `ease-out` 为主
- 不用过弹的 spring

## 启动台高保真说明
### 版面
- 首屏占据较大纵向空间
- Hero 与知识宇宙总览之间要有呼吸感，不要贴得过紧
- Continue 区应像“快速跳回入口”，视觉轻于总览区，重于卡片区

### 视觉焦点
启动台应有 3 个视觉焦点，按顺序：
1. Hero 输入框
2. 知识宇宙总览区
3. Continue 卡片

## 工作台高保真说明
### 版面
- 左栏较窄，偏导航
- 中栏最大，承载图谱
- 右栏固定宽度，承载阅读
- 底部输入区保持明显但不压迫

### 视觉焦点
工作台应有 3 个视觉焦点，按顺序：
1. 当前节点
2. 图谱结构
3. 底部输入区

## 响应式原则
### 平板
- 启动台星图缩小但保留
- 工作台右栏可折叠

### 手机
- 启动台保留 Hero 和卡片流
- 总览图改为简化缩略视图
- 工作台改为：
  - 顶部栏
  - 图谱区
  - 底部抽屉详情
  - 吸底输入

## 禁止项
- 紫色主视觉
- 高饱和渐变铺满整页
- 默认系统字体组合
- 实心厚重图标
- 重文案 onboarding
- 白底灰卡标准 SaaS 版式
- 过度玻璃拟态
- 复杂 3D 粒子背景

## 交付顺序建议
1. 先实现启动台 Hero + 总览区视觉
2. 再实现启动台卡片和筛选条
3. 然后实现工作台三栏和底部输入区
4. 最后补图谱节点样式、tooltip 和轻动效
